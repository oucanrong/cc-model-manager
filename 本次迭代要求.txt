程序运行正确，优化以下几处：

1. 选择API供应商和启动目标后，都要刷新代理设置，每个API供应商和启动目标都要对应不同的代理设置，单独刷新显示并且保存到config.json文件里，目前切换API供应商和启动目标时，不会刷新代理设置。

2. 更新claude code标签里的glm供应商和模型：

Claude Code 内部模型环境变量与 GLM 模型对应关系，默认配置如下：
ANTHROPIC_DEFAULT_OPUS_MODEL：GLM-4.7
ANTHROPIC_DEFAULT_SONNET_MODEL：GLM-4.7
ANTHROPIC_DEFAULT_HAIKU_MODEL：GLM-4.5-Air

```
{
  "env": {
    "CLAUDE_CODE_AUTO_COMPACT_WINDOW": "1000000",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "glm-4.5-air",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "glm-5.2[1m]",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "glm-5.2[1m]"
  }
}
```

注意开启 GLM 1M 上下文需要模型后缀加上 [1m] 即 glm-5.2[1m], 同时配置压缩窗口大小参数 "CLAUDE_CODE_AUTO_COMPACT_WINDOW": "1000000"
若模型参数加上 [1m] 后缀后 Claude Code 识别模型不存在，则需要升级 Claude Code 到最新版本后重试。
GLM-5.2/GLM-5-Turbo 作为高阶模型，对标 Claude Opus， 调用时将按照 “高峰期 3 倍，非高峰期 2 倍” 系数消耗额度。
作为限时福利，GLM-5.2/GLM-5-Turbo 将在非高峰期仅作为 1 倍抵扣，持续到 9 月底。注：“高峰期”为每日的 14:00～18:00 （UTC+8）。


如何切换 effort（思考强度）：在 Claude Code 会话中，输入 /effort 命令即可切换思考强度。
切换后，Claude Code 内部会将所选的 effort 映射到 GLM-5.2 实际生效的 effort，对应关系如下：


| Claude Code 中选择的 effort | GLM-5.2 实际映射的 effort |
| --- | --- |
| low, medium, high（默认值）| high |
| xhigh, max, ultracode | max |

💡 Coding 任务推荐切换至 max effort，以获取更深度的推理与更稳定的复杂任务表现。


3. claude code标签里的每一种模型，都要在界面和临时环境变量中加入以下变量：
